Exercício 9: Tratamento de Erros Mais Robusto

  • Descrição: Modifique um dos exercícios anteriores (por exemplo, o Exercício 2 ou 4) para simular um erro de rede (você pode fazer isso apontando para um URL inválido ou usando um setTimeout para introduzir um atraso e depois rejeitar a promise) e garantir que a mensagem de erro seja exibida de forma clara para o usuário.
    Exemplo de URL inválido: https://jsonplaceholder.typicode.com/invalid-url
  • Conteúdo Teórico:
    • O bloco catch em Promises (tanto com fetch quanto com axios) é essencial para lidar com erros de rede, erros de servidor (status de resposta 4xx, 5xx), ou erros de parsing.
    • É importante não apenas logar o erro no console, mas também atualizar o estado do componente para exibir uma mensagem amigável ao usuário.

Dica:
Para simular um erro, você pode mudar o URL da API para algo que não existe, como https://jsonplaceholder.typicode.com/invalid-url.
Sugestão de Extensão:
Implemente um botão "Tentar Novamente" que dispare a requisição novamente após um erro.
