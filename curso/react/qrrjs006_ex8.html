Exercício 8: Passando Funções como Props

  • Descrição: Crie um componente ContadorDeCliques que exiba um número e um botão. Crie outro componente BotaoIncrementar que receba uma prop onClick (uma função). Quando o botão for clicado, ele deve chamar a função passada via onClick.
  • Conceitos Teóricos:
    • Você pode passar funções como props para componentes filhos. Isso é essencial para a comunicação entre componentes, permitindo que componentes filhos "notifiquem" os pais sobre eventos.
    • No React, eventos (como onClick, onChange, etc.) são passados como atributos JSX, com o nome do evento em camelCase. O valor é uma função.

Exemplo de Código:

// App.js
import React, { useState } from 'react';

function BotaoIncrementar({ onClick }) {
  return (
    <button onClick={onClick}>
      Incrementar
    </button>
  );
}

function ContadorDeCliques() {
  const [contador, setContador] = useState(0);

  const handleClick = () => {
    setContador(contador + 1);
  };

  return (
    <div>
      <p>Cliques: {contador}</p>
      <BotaoIncrementar onClick={handleClick} />
    </div>
  );
}

function App() {
  return (
    <div>
      <h3>Contador:</h3>
      <ContadorDeCliques />
    </div>
  );
}

export default App;



Dicas:
O useState é um Hook do React que permite adicionar estado a componentes funcionais. Não se preocupe muito com ele agora, o foco é entender como as funções são passadas como props.
Sugestão de Extensão:
Adicione um botão "Decrementar" que também receba uma função via props para diminuir o contador.
