Exercício 9: Rotas Relativas e Absolutas

  • Descrição: Na sua página `Dashboard.js`, modifique os `Link`s para `Profile` e `Settings` para usarem caminhos relativos. Entenda a diferença entre caminhos relativos e absolutos em rotas.
  • Conceitos Teóricos:
    • Caminhos Absolutos: Começam com `/` e referem-se à raiz do aplicativo (ex: `/dashboard/profile`).
    • Caminhos Relativos: Não começam com `/` e são relativos ao caminho da rota pai. São úteis para rotas aninhadas, pois o componente pai já conhece sua base.

Exemplo de Código:

// src/components/Dashboard.js (atualizado)
import React from 'react';
import { NavLink, Outlet } from 'react-router-dom';

function Dashboard() {
  const activeLinkStyle = ({ isActive }) => ({
    fontWeight: isActive ? 'bold' : 'normal',
    color: isActive ? 'green' : 'purple',
  });

  return (
    <div>
      <h2>Dashboard</h2>
      <nav>
        <ul>
          <li><NavLink to="profile" style={activeLinkStyle}>Perfil</NavLink></li> {/* Caminho relativo */}
          <li><NavLink to="settings" style={activeLinkStyle}>Configurações</NavLink></li> {/* Caminho relativo */}
        </ul>
      </nav>
      <hr />
      <Outlet />
    </div>
  );
}

export default Dashboard;



Dicas:
- Em rotas aninhadas, o uso de caminhos relativos é geralmente mais robusto, pois o componente pai pode ser movido sem que as rotas filhas precisem ser atualizadas.
- Caminhos absolutos são úteis para navegação de qualquer lugar para uma rota específica.

Sugestão de Extensão:
- Crie um sub-sub-componente dentro de `Profile` e tente navegar para ele usando um caminho relativo do `Profile`.
