Exercício 1: Introdução à Context API - Criando um Contexto Simples

  • Descrição: Crie um contexto React chamado ThemeContext e um provedor (ThemeProvider) que envolva um componente filho. O contexto deve fornecer um valor simples (por exemplo, uma string 'light' para o tema). Exiba esse valor em um componente filho.
  • Conceitos Teóricos:
    • A Context API no React permite compartilhar dados que podem ser considerados "globais" para uma árvore de componentes, como tema, idioma do usuário ou dados autenticados.
    • createContext(): Cria um objeto Context. Quando o React renderiza um componente que se inscreve neste objeto Context, ele lê o valor de contexto mais próximo do Provider correspondente na árvore.
    • Context.Provider: É um componente React que permite aos componentes consumidores se inscreverem em alterações de contexto. Ele recebe uma prop value que é o valor a ser passado para os componentes descendentes que consomem este contexto.

Exemplo de Código:

// ThemeContext.js
import React, { createContext } from 'react';

export const ThemeContext = createContext('light'); // Valor padrão

// App.js
import React from 'react';
import { ThemeContext } from './ThemeContext';
import DisplayTheme from './DisplayTheme';

function App() {
  return (
    <ThemeContext.Provider value="dark">
      <DisplayTheme />
    </ThemeContext.Provider>
  );
}

// DisplayTheme.js
import React, { useContext } from 'react';
import { ThemeContext } from './ThemeContext';

function DisplayTheme() {
  const theme = useContext(ThemeContext);
  return <h2>Tema atual: {theme}</h2>;
}



Dica:
Lembre-se de importar createContext e useContext dos seus respectivos locais. A hierarquia dos componentes é crucial para que o provedor envolva o consumidor.

Sugestão de Extensão:
Crie um botão que, ao ser clicado, altere o valor do tema para 'light' ou 'dark' dentro do ThemeProvider.
