Exercício 4: Inserindo Dados (INSERT)

  • Descrição: Crie uma API para adicionar novos produtos à tabela produtos. A API deve aceitar requisições POST na rota /produtos com um corpo JSON contendo nome e preco do produto.
  • Conceitos Teóricos:
    • Rotas POST: Usadas para enviar dados a um servidor para criar ou atualizar um recurso.
    • req.body: No Express, o corpo da requisição (geralmente JSON para APIs) é acessado através de req.body. Para isso funcionar, você precisará de um middleware para parsear JSON, como express.json().
    • SQL INSERT INTO: A instrução SQL para adicionar novas linhas a uma tabela.
    • Middleware express.json(): É um middleware embutido no Express que analisa requisições com corpo JSON e as disponibiliza em req.body.

Dicas:
Use uma ferramenta como Postman, Insomnia ou a extensão "REST Client" do VS Code para testar requisições POST.
O código de status HTTP 201 Created é o ideal para uma inserção bem-sucedida.
Sugestão de Extensão:
Adicione validação para garantir que o preco é um número válido.
