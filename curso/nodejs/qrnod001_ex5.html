Exercício 5: Lidando com Requisições POST

  • Descrição: Crie uma rota /enviar-dados que aceite requisições POST. Quando uma requisição POST é feita para essa rota, o servidor deve responder com a mensagem "Dados recebidos com sucesso!". Por enquanto, não se preocupe em ler os dados enviados no corpo da requisição.
  • Conteúdo Teórico:
    • Requisições POST são usadas para enviar dados para o servidor, geralmente para criar ou atualizar recursos.
    • app.post(path, callback) é o método do Express para lidar com requisições POST.
    • Para testar requisições POST, você precisará de uma ferramenta como o Postman, Insomnia, ou a extensão Thunder Client no VS Code, ou até mesmo um formulário HTML básico.

Dica:
Lembre-se de iniciar seu servidor e usar uma ferramenta externa para fazer a requisição POST. Um navegador web por padrão faz requisições GET.
