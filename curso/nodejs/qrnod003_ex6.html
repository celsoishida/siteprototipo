Exercício 6: Atualizando Dados (UPDATE)

  • Descrição: Crie uma API para atualizar as informações de um produto existente. A API deve aceitar requisições PUT na rota /produtos/:id com um corpo JSON contendo nome e preco (ambos opcionais, mas pelo menos um deve ser enviado).
  • Conceitos Teóricos:
    • Rotas PUT: Usadas para atualizar um recurso existente completamente, ou para criar se não existir (embora para APIs RESTful, POST é mais comum para criação e PUT/PATCH para atualização).
    • SQL UPDATE: A instrução SQL para modificar dados existentes em uma tabela.
    • Condição WHERE em UPDATE: Crucial para garantir que você atualize apenas o registro desejado.

Dicas:
Teste a atualização de apenas o nome, apenas o preço, e ambos.
Verifique o affectedRows do resultado da query para saber se algum registro foi realmente atualizado.
Sugestão de Extensão:
Em vez de PUT, implemente uma rota PATCH para atualização parcial, que seria mais adequada.
