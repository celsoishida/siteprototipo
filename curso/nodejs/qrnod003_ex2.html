Exercício 2: Listando Dados (SELECT) com Parâmetros

  • Descrição: Modifique a API do exercício anterior para que ela possa listar um produto específico por seu id. A rota deve ser /produtos/:id.
  • Conceitos Teóricos:
    • Parâmetros de Rota: O Express permite capturar valores dinâmicos na URL usando : (dois pontos). Por exemplo, /produtos/:id significa que :id será um parâmetro.
    • SQL WHERE: A cláusula WHERE é usada para filtrar registros que atendem a uma condição especificada.
    • Prevenção de SQL Injection: É crucial usar prepared statements (declarações preparadas) com ? (interrogação) para passar parâmetros para suas consultas SQL. Isso evita ataques de injeção de SQL.

Dicas:
Teste a rota com IDs existentes e inexistentes para ver o comportamento.
Lembre-se de sempre validar os dados de entrada, mesmo que de forma simples.
Sugestão de Extensão:
Adicione uma rota /produtos/categoria/:nomeCategoria para buscar produtos por categoria.
