Exercício 3: Listando Dados (SELECT) com Parâmetros e Nulos

  • Descrição: Crie uma rota GET /pedidos que permita filtrar pedidos por cliente_id e/ou status. Se nenhum parâmetro for fornecido, deve listar todos os pedidos. Se um dos parâmetros for null ou não for fornecido na query string, ele deve ser ignorado na filtragem.
  • Conceitos Teóricos:
    • Query Parameters: Parâmetros de consulta são passados na URL após ? e são pares chave-valor (ex: /pedidos?cliente_id=1&status=pendente). No Express, eles são acessíveis via req.query.
    • Construção Dinâmica de Consultas SQL: Às vezes, você precisará construir sua consulta SQL de forma dinâmica com base nos parâmetros recebidos.
    • Valores Nulos (NULL) em SQL: Em bancos de dados, NULL representa a ausência de valor. Você pode verificar se um campo é NULL usando IS NULL.

Dicas:
Crie uma tabela pedidos com colunas id, cliente_id, status (ex: 'pendente', 'finalizado') e data.
Teste as seguintes URLs: /pedidos, /pedidos?cliente_id=1, /pedidos?status=pendente, /pedidos?cliente_id=2&status=finalizado.
Sugestão de Extensão:
Adicione um filtro por data_inicial e data_final para buscar pedidos em um intervalo de datas.
