Exercício 5: Inserindo Dados (INSERT) com Parâmetros Opcionais e Nulos

  • Descrição: Crie uma API para inserir novos usuários na tabela usuarios. A API deve aceitar requisições POST na rota /usuarios com corpo JSON contendo nome, email e idade (opcional). Se a idade não for fornecida, ela deve ser inserida como NULL no banco de dados.
  • Conceitos Teóricos:
    • Parâmetros Opcionais: Nem todos os campos de um registro são obrigatórios. Sua API deve ser flexível para lidar com a ausência de alguns dados.
    • Valores Padrão e NULL: No SQL, você pode definir valores padrão para colunas, ou permitir que sejam NULL. Sua lógica de API deve refletir isso.

Dicas:
Crie a tabela usuarios com colunas id, nome (VARCHAR), email (VARCHAR UNIQUE), idade (INT, NULLABLE).
Teste enviando requests com e sem o campo idade.
Sugestão de Extensão:
Implemente uma validação simples de e-mail usando uma expressão regular.
