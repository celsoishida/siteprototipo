Exercício 7: Middleware para Autenticação Simples

  • Descrição: Crie um middleware que verifique se um header Authorization está presente na requisição. Se estiver, chame next(). Caso contrário, retorne um status 401 (Unauthorized) com a mensagem "Acesso não autorizado.". Aplique este middleware a uma rota específica, por exemplo, /secreta.
  • Conteúdo Teórico:
    • Headers HTTP são pares nome-valor que contêm informações contextuais sobre a requisição ou resposta.
    • Você pode acessar os headers da requisição através de req.headers. Por exemplo, req.headers.authorization.
    • res.status(statusCode) define o status HTTP da resposta.
    • res.send() ou res.json() envia o corpo da resposta.

Exemplo de Código:

const verificaAuth = (req, res, next) => {
  const token = req.headers.authorization;
  if (token) {
    console.log('Token de autorização presente.');
    next();
  } else {
    res.status(401).send('Acesso não autorizado. Token de autorização ausente.');
  }
};
app.get('/secreta', verificaAuth, (req, res) => {
  res.send('Bem-vindo à página secreta!');
});



Dica:
Para testar, use uma ferramenta como Postman e adicione um header Authorization com qualquer valor.
