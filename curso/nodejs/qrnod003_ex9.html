Exercício 9: Deletando Dados (DELETE) com Parâmetros Opcionais e Nulos

  • Descrição: Crie uma API para "cancelar" pedidos. A API deve aceitar requisições DELETE na rota /pedidos/:id. Para fins deste exercício, "cancelar" significa mudar o status do pedido para 'cancelado' e, opcionalmente, definir data_cancelamento para a data/hora atual.
  • Conceitos Teóricos:
    • "Soft Delete" ou Mudança de Status: Em muitos cenários, é melhor marcar um registro como inativo ou mudar seu status em vez de excluí-lo permanentemente. Isso preserva o histórico.
    • Funções SQL de Data/Hora: Usar funções como NOW() ou CURRENT_TIMESTAMP() para registrar a data e hora atuais no banco de dados.

Dicas:
Adicione uma coluna data_cancelamento (DATETIME, NULLABLE) à sua tabela pedidos.
Teste a API em um pedido já cancelado para ver a mensagem de erro.
Sugestão de Extensão:
Permita que a API aceite um parâmetro de query opcional force_delete=true para remover o registro permanentemente.
