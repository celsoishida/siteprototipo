Exercício 8: Deletando Dados (DELETE)

  • Descrição: Crie uma API para deletar um produto existente por seu id. A API deve aceitar requisições DELETE na rota /produtos/:id.
  • Conceitos Teóricos:
    • Rotas DELETE: Usadas para remover um recurso do servidor.
    • SQL DELETE FROM: A instrução SQL para remover linhas de uma tabela.
    • Cuidado com DELETE: É uma operação destrutiva, então a validação e a confirmação são importantes em aplicações reais.

Dicas:
Use o status HTTP 200 OK (ou 204 No Content se não houver corpo de resposta) para uma exclusão bem-sucedida.
Sempre verifique affectedRows para saber se a exclusão realmente ocorreu.
Sugestão de Extensão:
Implemente uma "soft delete" onde um campo ativo ou deletado_em é atualizado em vez de remover o registro.
