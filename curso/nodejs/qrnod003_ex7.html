Exercício 7: Atualizando Dados (UPDATE) com Parâmetros Opcionais e Nulos

  • Descrição: Crie uma API para atualizar parcialmente as informações de um cliente. A API deve aceitar requisições PATCH na rota /clientes/:id com um corpo JSON contendo nome, email e/ou telefone. Qualquer campo não fornecido deve ser ignorado. Se um campo for fornecido com null, ele deve ser atualizado para NULL no banco de dados.
  • Conceitos Teóricos:
    • Rotas PATCH: Semelhantes a PUT, mas usadas para aplicar modificações parciais a um recurso.
    • Lógica de Atualização Condicional: Construir dinamicamente a consulta SQL com base nos campos presentes no req.body e tratar explicitamente valores null.

Dicas:
Crie uma tabela clientes com id, nome, email, telefone (VARCHAR, NULLABLE).
Teste enviando { "email": "novo@email.com" } e { "telefone": null } para o mesmo cliente.
Sugestão de Extensão:
Adicione validação para o formato de email e telefone, se aplicável.
